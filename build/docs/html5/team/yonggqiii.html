PROJECT: UniLA

Overview

UniLA is a desktop utilities application designed for NUS students who are typing oriented. UniLA provides an efficient and convenient solution for managing contact list and event list, contact interactions, planning meetings, setting up reminders, etc. The application is primarily concerned with CLI (Command Line Interface) Interaction, with a simple and intuitive GUI provided.

Planning a meeting : meet

Creates a meeting event among certain people. Currently, an event will be created at the earliest 2h timeslot starting from the time the command is run. Default meeting names, venues and labels are provided at this stage. Not more than one meet event created by this tool can exist in the application as of v1.3, due to restrictions in event naming and so forth. Subsequent versions would allow for Format: meet INDEX

  • INDEX must be positive integers 1,2,3…​

  • INDEX may take on multiple indices.

Examples:

  • list
    List all contacts in index order, find the contact index.
    meet 1 5 8
    Create a suitable meeting timeslot and venue among the first, fifth and eighth person.

Switching views: switch

Changes the view in the window. The default upon startup is in Contacts view. Running this command will toggle between contacts and events view. Format: switch

Commands that require indices of one particular view as arguments may require you to switch to that view first. In those cases, switch to the appropriate view before running the command.

Meeting Planner

The mechanism is supported by the MeetCommand class. It allows users to create and store meeting events by automatically finding suitable timeslots to meet.

Given below is an example usage scenario and how the meet command behaves.

Step 1. The user lists people stored in the addressbook.

Step 2. The user runs the command meet 1 4 5, which indicates that the user wants to form a meeting with the 1st, 4th and 5th people listed.

Step 3. A list of possible events and the center location will be shown for the user to add.

How it works:

  • MeetCommandParser parses meet command. The MeetCommandParser will ensure that positive integer indices are entered in the CLI. If the command does not meet the requirements, a ParseException will be thrown. The parser also removes duplicate indices by using the java.util.Set interface.

  • Indices are passed into MeetCommand for retrieval. The indices entered will be passed into the MeetCommandParser.

  • MeetCommand retrieves persons from UniLA. The persons retrieved will be stored temporarily in the instance.

  • Execute will pull the entire events list from the UniLA.

  • The events will be piped through a stream and filtered such that all events contain at least one of the people listed to meet.

  • The first 2h timeslot starting from the time the command was entered would be found.

  • Event will be automatically created, added to the model, and set as the selected event, and the event list will be shown in the UI.
